How I write software with LLMs

LLM编程经验

📝 简介

作者 Stavros 分享了他日常使用 LLM 写代码的实战经验，包括最佳实践和踩坑总结。这篇文章详细描述了他如何使用 AI 辅助编程，包括使用的工具、工作流程、以及真实案例。

🎯 核心观点

  • LLM 可以显著降低代码缺陷率，甚至低于手写代码
  • 工程师的技能从"写正确代码"转变为"正确架构系统"
  • 不同人使用 LLM 的结果差异很大，对话方式很重要
  • 代码审查粒度在变化：从逐行 → 函数级 → 架构级

🛠️ 他用 LLM 构建的项目

  • Stavrobot - 一个注重安全的 AI 助手（类似 OpenClaw）
  • Middle - 语音备忘录 pendant硬件 + 转录
  • Sleight of Hand - 艺术作品：不规则跳动的时钟
  • Pine Town - 多人在线绘画画布

⚙️ 工作流程

作者使用 OpenCode 作为开发框架，采用"架构师-开发者-评审"三人协作模式：

  • 架构师（Opus 4.6）- 只与人交互，制定详细计划
  • 开发者（Sonnet 4.6）- 根据计划实现代码
  • 评审（多模型）- Codex/Gemini/Opus 独立评审

关键点：必须说 "approved" 才会开始执行

💡 关键经验

  • 使用多模型 - 不同模型有不同优缺点，组合使用效果最佳
  • 有数据的日期才能点击 - 无数据的日期显示为灰色
  • agent 需要相互调用 - 避免繁琐的信息传递
  • skill 文件手写 - 让 LLM 写指令效果不好

⚠️ 失败模式

当你对底层技术不够熟悉时，无法 catch LLM 的错误决策，导致在错误基础上越堆越多，最终无法收拾。

解决：规划阶段尽量理解技术细节，好好引导 LLM
